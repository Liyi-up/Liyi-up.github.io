一、JSR-107缓存规范

1、核心接口

  1. CachingProvider(缓存提供者):定义创建、配置、获取、管理和控制多个CacheManager。一个应用可以在运行期访问多个CachingProvider。
  2. CachManaer(缓存管理器):定义创建、配置、获取、管理和控制多个唯一命名的Cache，这些Cache存在于CacheManaer的上下文中。一个CacheManager仅被一个CachingProvider所拥有。
  3. Cache(缓存组件):是一个类似Map的数据结构并临时存储以key为索引的只，一个Cache仅被一个CacheManager所拥有。
  4. Entry(记录):是一个存储在Cache中的key-value对
  5. Expriy:(有效期)每一个存储在Cache中的条目有一个定义的有效期，一旦超过这个时间，条目为过期状态。一旦过期，条目将不可访问、更新和删除。缓存有效期可以通过ExpiryPolicy设置。

    • @CacheConfig(cacheNames="Good") //抽取缓存的公共配置
public interface GoodsService {
    /**
     *   @Cacheable 将方法的运行结构进行缓存，下次使用相同的数据直接从缓存中获取
     *   属性:
     *      CacheNames/value:缓存的名字；将方法的返回结果放在哪个缓存中，是数组的方式，可以指定多个缓存
     *      key:缓存数据使用key，可以指定，不指定默认就是方法参数的值 1-方法返回的值
     *          编写使用spEl表达式 :#root.methodName=方法名 ; #id；方法参数id的值 #a0 #p0=#root.args[0]
     *      keyGenerator：key的生成器；可以自己指定key的生成器的组件id
     *      key/keyGenerator只能二选一；指定好keyGENERATOR
     *      cacheManager:指定缓存管理器；或者cacheResolver缓存解析器；作用相同
     *      condition:指定符合条件的情况下才缓存
     *      unless:否定缓存，当unless指定的条件为true，方法的返回值就不会被缓存；可以获取到结果进行判断
     *      sync:是否使用异步，默认false
     */
    //查询
    List<Goods> getGoods();

    //单个查询
//    @Cacheable(cacheNames = "good",key = "#root.methodName+'['+#id+']'")
    @Cacheable(key = "#id")
    Goods getGood(Integer id);
/**
 * @CachePut 即调用方法又更新缓存数据;
 * 修改了数据库的某个数据，同时更新缓存;
 * 1.先调用目标方法
 * 2.将目标方法的结果缓存起来
 *
 * 测试步骤:
 * 1.查询1号商品；查到结果放在缓存中;
 * 2.以后查询还是之前的结果
 * 3.跟新1号商品:
 *      将方法的返回值也放进缓存了
 * 4.查询1号商品?
 * 应该是更新后的员工，但是查询出来的结果仍然是没有更新的
 * 原因:因为我们的两个方法的缓存key不统一，所以更新之后并没有在原来的缓存中也进行一次更新
 * key = "#good.goodsId"指定传入时good的id或result.id
 * 注意:@Cachable的key不能使用#result的 原因:因为我们在方法运行前就需要得到key
 * 并且目标方法的返回值也必须一致
 * value/CacheName还必须是同一个Cache组
 */
    //更新
    @CachePut(key = "#good.goodsId")
    Goods updateGood(Goods good);
/**
 * @CacheEcict: 缓存清除
 *key：指定要清除的数据
 * allEntries 是否删除指定缓存组中的所有数据
 * beforeInvocation=false;缓存的清除是否在方法之前执行
 * 默认代表缓存是在方法之后执行；如果目标方法出错缓存就不会清除
 */

    //删除
    @CacheEvict(allEntries = true)
    int DelGood(Integer id);

    //新增
    int AddGood(Goods good);

    //根据商品类型来数据
    //@Caching定义复杂的缓存注解
    @Caching(
            cacheable = {
                    @Cacheable(key = "#type")

            },
            put = {
                    @CachePut(key = "#result.goodsId"),
                    @CachePut(key = "#result.goodsType")
            }
    )
    Goods getByType(String type);

}

    • SimpleCacheConfiguration;给容器中注册一个ConcurrentMapCacheManager类型的cacheManager

    • @Configuration(
    proxyBeanMethods = false
)
@ConditionalOnMissingBean({CacheManager.class})
@Conditional({CacheCondition.class})
class SimpleCacheConfiguration {
    SimpleCacheConfiguration() {
    }

    @Bean
    ConcurrentMapCacheManager cacheManager(CacheProperties cacheProperties, CacheManagerCustomizers cacheManagerCustomizers) {
        ConcurrentMapCacheManager cacheManager = new ConcurrentMapCacheManager();
        List<String> cacheNames = cacheProperties.getCacheNames();
        if (!cacheNames.isEmpty()) {
            cacheManager.setCacheNames(cacheNames);
        }

        return (ConcurrentMapCacheManager)cacheManagerCustomizers.customize(cacheManager);
    }
}

5、运行流程(@Cacheable为例):

  1. 目标方法执行前,先根据CacheNames去查询Cache组件

    (CacheManager先获取相应的缓存)，第一次获取,如果没有缓存会自动创建,

  2. 去Cache中查找缓存的内容，使用key，默认是使用keyGenerator生成，默认使用SimpleKeyGenerator生成key

    SimpleKeyGenerator生成key的默认策略：

    • 如果没有参数:key=new SimpleKey()
    • 如果有一个参数:key=参数的值;
    • 如果有多个参数key = new SimpleKey(params);

  3. 没有查到缓存则调用目标方法，将目标方法返回的结果放到缓存中

  4. 总结:@Cacheable标注的方法执行之前先来检查缓存中有没有这个数据，按照默认参数的key去查询缓存，如果没有就运行方法，并将结果存储到缓存中。再次调用即可直接使用缓存中的数据

  5. 核心:

    • 使用CacheManager[ConcurrentMapCacheManager]按照名字得到Cache[ConcurrentMapCache]组件.
    • key是使用KeyGenerator，默认是SimpleKeyGenerator
