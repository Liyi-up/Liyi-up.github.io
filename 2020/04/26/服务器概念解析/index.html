开发中常见服务器概念的解析

一、什么是服务器?

​ 服务器的硬件其实好理解,其实就是一台性能、稳定性、扩展性比我们普通个人PC要强的一台机器而已，它也需要搭载操作系统，比如有专windows server，或者各种各样的linux发行版本操作系统，目前一些中小型企业都已经开始将服务器架设到云端(云服务器)，总而言之只要是运行在操作系统之上，绑定了服务器特点的IP地址，并且在某一个端口监听用户的请求，提供相应服务的软件，我们都可以称之为服务器软件。

二、“静态服务器”与“动态服务器”

​ 先从字面上来理解，提供静态服务的服务器我们称之为静态服务器，提供动态服务的服务器我们称之为动态服务器，那什么又是静态服务？动态服务呢？其实很好理解，像本人的这个博客就是典型的静态博客，静态服务一般返回的就是静态资源，是固定的，比如无论任何人，任何时间，访问我的这个个人博客所获取到的内容都是一模一样的 ，而动态服务一般返回的就是动态资源，典型的场景，比如用户去登录，一个网站的后台，比如用户登录B战，服务器会根据你登录的身份信息和权限信息，去数据库中动态的获取相应的内容和资源，不同的人看到的B站页面是不一样的，这种场景在现时生活中比比皆是。

三、Web服务器、HTTP服务器、以及应用服务器的概念。

​ 1.Web服务器:

​ Web服务器的概念范围比较广，故名思意，Web服务器的功能就是提供Web化服务，那什么又叫Web化服务呢？简单的理解，比如说:网站页面服务，邮件服务、一些网络下载的服务等等，我们都可以称作Web化服务，因此，Web服务器从广义上来说，就是相应用户的需求，然后提供服务，所以从这个角度上来讲，当下几乎所有的服务器软件，我们都可以称之为Web服务器。

​ 2.HTTP服务器:

​ 由于当今Web服务器应用层通讯协议主就是HTTP协议，所以基本上可以将HTTP服务器理解为和Web服务器对等，HTTP服务器只是把服务器商店资源通过HTTP协议形式传输给客户端，所以一般侧重于静态文件传输的支持，因此我们一般也可以把它理解为静态服务器。

​ 3.应用服务器:

​ 应用服务器我们可以理解为一个特定应用的承载容器，一般来说需要有运行时环境的支持，比如Java领域比较常用的应用服务器Tomcat，它就必须要有java的runtime运行时环境支持，因此应用服务器也常常被称作动态服务器，动态容器、应用容器、Web容器等等，既让属于动态服务器，那么很明显它就支持动态响应，比如可以根据不同的用户请求，能够动态的生成资源，并且返回给用户，这样一来不同用户拿到的返回内容也就不一样，它不像静态服务器，大家拿到的资源都一样，一般来说，在流量不打的场景下，应用服务器，应用服务器肯定也是可以做HTTP服务器来用的，只不过很少有人会这么去做。

四、目前市场流行的几款应用服务器软件

​ 1.Nginx：

Nginx是一个典型的HTTP服务器，它的本职工作就是将服务端的静态内容或静态资源，通过HTTP协议的形式传输给客户端，所以它就是典型的静态服务器，当然Nginx的功能远远不止这些,目前常用的功能有反向代理、负载均衡等等，在现实的应用部署场景中，Nginx一般是和后面真正的动态服务器打配合，比如Tomcat，把用户发送的请求进行转发给真正的Web服务器，从而提供灵活稳定的Web服务器，因为Nginx服务器的性能特别好，稳定性也特别高，能够抗住冲击，所以我们一般把它放到最前面，去直面用户请求。)当然Nginx服务器本身可以在诸如Lua脚本等辅助下做二次开发，可以编程一个提供动态服务的应用服务器(OpenResty)。

Nginx是纯C语言编写的服务器，消耗小，性能高，这些优点让它成为目

前市面上大量的互联网公司在重度使用Nginx。

2.Tengine:

Tengine其实是基于Nginx做的改造，)我们可以看作是在Nginx的基础上又做了加强和封装，主要是针对大流量网站的场景，添加了很多的高级功能和特性。

Tengine毕竟是阿里的项目，最先是由淘宝发起的，所以它的性能和稳定性都得到了很好的实际检验，所以Tengine，它和Nginx哥俩是一个性质的东西。

3.Apache:

Apache HTTP Server，它是Apache软件组织旗下的一个HTTP服务器项目，Apache从功能上来说，也是和Nginx服务器对等的，也是做静态HTTP服务器居多，只不过在并发性、负载性、以及资源消耗方面，性能其实比不上Nginx的，所以目前企业开发选择Nginx的居多。

4.IIS:

IIS是微软阵营的一个的一个Web服务器，只能运行在Windows系统下，而且它并不开源，它的主要身份，其实也是一个HTTP静态服务器，但它也支持像ASP.NET这种类似环境的应用，因为都是微软旗下的项目，因此也可以称作是一个应用服务器来用，所以它可以定位为一个具有应用服务器能力的HTTP服务器。

5.Tomact:

Tomcat和上面的Apache HTTP Server一样，都是隶属于Apache软件组织旗下的一个子项目，但是Tomcat它却是一个典型的javaEE应用服务器软件，或者说是动态容器、Web容器等等，提供的是动态服务，，而且需要java运行时环境支持，所以它和上面讲过的Nginx、Tengine、Apache等等不是一个性质的服务器软件，众所周知，现在大名鼎鼎的Spring Boot就将Tomacat作为内嵌的默认应用服务器，Tomcat本事其实是一个也仅仅是一个支持Servlet标准的Java EE应用容器，所以主要用途也就是做Java EE应用服务器，虽然Tomact是一个动态应用服务器，但是在流量不大的场景里面当然也可以直接做HTTP服务器来使用，但是一般不会这样用，不会让Tomcat去直面客户，而是像上面提到的和Nginx这种高性能的静态服务器打配合，Niginx在前面负责接收用户请求，然后转发到后面真正的Tomcat容器去处理。

6.Jetty:

Jetty也是SpringBoot框架支持的内嵌应用容器之一，只不过不是默认的而已，默认是Tomcat，所以Jetty和Tomcat是一个性质的东西，按照Jetty官网来说Jetty是一个开源的HTTP服务器+应用容器，因此既可以提供静态服务，也可以提供动态服务，而且提供了对HTTP/2、WebSocket、jmx等等许多其他集成的这种支持，如果拿Jetty和Tomcat对比，两者在功能上几乎是一致的都是做动态应用服务器比较多，都符合Servlet标准，只是Jetty比Tomcat要轻量一些，配置上也会跟简单一些而已。

7.Undertow:

Undertow和上面的jetty、Tomcat一样也是大名鼎鼎的Spring Boot所支持的内嵌应用容器之一，当然也不是默认的，Undertow是Red Hat 红帽子公司开源的一款基于java语言编写的应用服务器，也是Wildfly应用服务器的的默认的Web容器，Undertow也符合Servlet标准规范，支持HTTP/2、WebSocket也是一款灵活的高性能的Web应用服务器，在高并发量不高的情况下其实Undertow和Tomcat、Jetty性能区别不大，但是在并发量稍微高一点的情况下，其实Undertow的性能，要优于Jetty和Tomcat。

8.JBoss和Wildfly(重量级)：

JBoss也是一个Java EE应用服务器，可以部署动态应用，但它不仅仅是所谓的Servlet应用容器，而且是EJB应用容器，JBoss在早年的确是非常火的一款动态应用服务器软件，尤其是在早年的EJB时代，发挥了巨大的作用，而且更广泛一样上的JBoss不仅仅是一个java应用服务器了，而是一个全套解决方案，所以也算是一个比较重量级别的应用服务平台JBoss AS从版本8开始更名为WildFly了，因此现在如果视图去访问JBoss官网的话，JBoss官网会提示现在已经迁移到Wildfly了，所以现在取而代之的WildFly也是一款企业级的应用服务器，而且它是由Red Hat红帽子公司开发维护的，而且WildFly应用服务器钟的默认Web容器就是上面讲过的Undertow。

9.WebLogic:

WebLogic是Oracle公司出品的，用于部署企业级Java应用的一个应用服务器，它可以说是一个全能型的重量级别的应用服务器软件，因为支持Java EE几乎所有的应用规范包括什么EJB、Servlet、Jms….等等，当然了它毕竟是一个商业软件，有着强大的这种商业化和周边组件的支持和Tomcat这种开源免费的应用容器体量当然是不一样的。

10.WebSphere:

WebSphere是IBM公司出品的一个Java EE应用服务器软件，功能非常强大，而且有着IBM的开发工具相配套，当然它也是一个价格不菲的商业服务器软件和WebLogic一样也是一个支持更多Java EE标准和规范的综合应用服务器，可靠性高，体量也会更大，性能更好、功能更全，当然相应的使用成本也会高很多。
