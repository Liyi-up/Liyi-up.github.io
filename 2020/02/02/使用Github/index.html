1.目的

借助github托管项目代码，通过github实现多人协同开发项目。

2.基本概念

  • 仓库(Repository)

​ 仓库的意思，即你的项目，想在Github上开源一个项目，那就必须要新建一个Repository，如果你开源的项目多了，你就拥有了多个Repository。

  • 收藏(Star)

​ 收藏项目，方便之后的查看。

  • 复制克隆项目(Fork)

​ 相当于你把别人的项目原封不动的复制一个独立存在的的到自己的GitHub上，系统并且会为你自动创建一个Repository，并且你可以在这个Fork过来的项目上进行继续开发更新的操作。

  • 发送请求(Pull Request)

​ 发送请求，其实是一个基于Fork的操作，即你在Fork一个别人的仓库过来后，你在这个项目上发生了一些改动更新，那么你可以通过Pull Request(PR)发送一个请求到原有的项目创建人那里，主人看到后如果满意便会把你的更新的地方合并的原始仓库中,这个时候你的改动便会出现在原始仓库中了。

  • 关注(Watch)

​ 这个玩过微博，抖音的一些社交软件的朋友都应该好理解，即你Watch了某个项目，以后只要这个项目有任何更新，你都会第一时间收到关于这个项目的通知提醒。

  • 事务卡片(lssue)

​ 发现代码BUG，但是目前没有成型代码，需要讨论时用,这里可以理解为你在公司当老板你发现了手下员工出现了一些工作问题，但你现在又不知道该怎么解决，这个时候你就可以lssue召开一次会议和大家讨论怎么去解决你发现的BUG。

  • Github主页

​ 账号创建成功或者点击网址导航栏github图标进入的页面即为Github主页，主页左侧主要显示用户动态及关注的用户或者仓库的动态，右侧显示所有git库。

  • 仓库主页

​ 仓库主页主要显示项目的信息，如：项目代码，版本，收藏/关注/fork情况等。

  • 个人主页

​ 个人信息:头像，个人简介，关注我的人，我关注的人，我关注的git库，我的开源项目，我贡献的开源项目等信息。

3.注册Github账号

  1. 因为github在国外服务器所以访问较慢或者无法访问，需要翻墙(Shadowsocks)

  2. 私有仓库只能自己或指定朋友才能使用

  3. 新注册的用户需要验证邮箱后才能使用
